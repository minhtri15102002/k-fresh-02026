## How to use this example

### As a Track P learner

1. Read the corresponding template first ([`templates/manager/README.md`](../../../../templates/manager/README.md))
2. Read this filled example to see the *level of specificity* a real filled artifact needs
3. Fill your own version in `training/sandbox/<your-name>/manager/` — **not** by copying this; by re-thinking the questions for your real (or hypothetical) team
4. PR your filled set using the title pattern from each module (e.g. `Track P · M5 — Capstone: Quality Org Charter v1.0`)

### As a trainer running a cohort

- Assign cohort members 1–2 artifacts to "deeply audit" — what would they change about Phoenix's? Why? What does that reveal about their own org's norms?
- Use Phoenix's **anti-principles** as a calibration anchor — most learners' first-draft anti-principles are weaker than these; the gap teaches the lesson.
- Phoenix's **decision-rights** table has 8 rows; spec only requires 6. Discuss which categories cohort members would add or remove for their own contexts.

### As a portfolio-builder for interviews

- Phoenix-style is a **director-tier** portfolio (M2 / D level). Most interviewers will recognise the structure.
- The single most differentiating piece is the **Quality Org Charter** — most candidates don't have one. Bring it.

## Generating the charter for your own team

The charter you see in [`quality-org-charter.md`](./quality-org-charter.md) was emitted by:

```bash
npm run charter:draft -- .agents/skills/quality-org-charter/resources/inputs-example.json \
  --out training/sandbox/example/manager/quality-org-charter.md
```

To produce one for your own team:

```bash
cp .agents/skills/quality-org-charter/resources/inputs-example.json /tmp/charter-inputs.json
$EDITOR /tmp/charter-inputs.json   # change every value to YOUR team's reality
npm run charter:draft -- /tmp/charter-inputs.json \
  --out training/sandbox/<your-name>/manager/quality-org-charter.md
```

See [`.agents/skills/quality-org-charter/SKILL.md`](../../../../.agents/skills/quality-org-charter/SKILL.md) for the full schema and validation rules.
