## Phase outcomes

You can:

- Explain the difference between QA, QC, and Testing.
- Map a feature request onto SDLC stages and identify QA touchpoints in each.
- Choose the right test level (unit / integration / E2E) for a given risk.
- Apply equivalence partitioning, boundary value analysis, decision tables, and state-transition testing to derive test cases.
- Author a test case that is **self-contained**, **deterministic**, **traceable**, and **review-ready**.
- File a defect report that a developer can reproduce on the first try.
